# 1. Общие положения
## 1.1. Назначение
Программная система предназначена для выполнения алгебраических операций над полиномами, хранящимися в таблицах под уникальными именами.
## 1.2. Область применения
Система может быть использована в образовательных целях для изучения алгебры, а также в научных исследованиях, где требуется работа с полиномами.

# 2. Требования к системе
## 2.1. Функциональные требования
### Хранение полиномов
- Полиномы должны храниться в виде списка.
- Каждый полином должен иметь уникальное имя (ключ).
- Все полиномы должны храниться одновременно в 6 различных структурах данных (таблицах):
  - Линейная таблица на массиве
  - Линейная таблица на списке
  - Упорядоченная таблица на массиве
  - Дерево (АВЛ или красно-черное)
  - Хеш-таблица с открытой адресацией
  - Хеш-таблица со списками

### Операции над отдельным полиномом
- Вычисление значения полинома в заданной точке (x, y, z).
- Умножение полинома на константу.
- Вычисление частной производной полинома по каждой из переменных (x, y, z).
- Вычисление интеграла полинома по каждой из переменных (x, y, z).

### Операции над выражениями из полиномов
- Сложение полиномов.
- Вычитание полиномов.
- Умножение полинома на константу.
- Умножение полиномов.
- Деление полиномов.
  
### Операции над таблицами
- Добавление нового полинома во все таблицы одновременно.
- Удаление полинома из всех таблиц одновременно по его имени.
- Поиск полинома в активной таблице по имени (используется при вычислении выражений, составленных из имён полиномов).

### Интерфейс пользователя
- Возможность выбора активной таблицы для отображения и операций.
- Таблица должна выводиться на экран в формате, как минимум, двух столбцов: 1) имя полинома, 2) строковое представление полинома.
- Ввод выражений для вычисления.
- Вывод результатов вычислений.
- Сообщения об ошибках (например, при попытке деления на ноль, отсутствии полинома в таблице и т.д.).

### Примеры
1. Исходные данные:
   
   Пусть в таблице хранятся следующие полиномы:
   
| Имя    | Полином                               |
| :----- | :------------------------------------ |
| pol1   | 3.2x^2y^3z^1 – 1.3x^1z^4              |
| pol2   | –3.2x^2y^3z^1 + 1.3x^1z^4             |
| const6 | 6.0                                   |
| q      | 4.0x^2                                |

2. Ввод выражения:
   
   Пользователь вводит следующее выражение для вычисления: \
   __new_pol = 2 * pol1 + 2 * pol2 + 3.6 * q – const6__

3. Обработка и вывод:

- Находит в активной таблице полиномы __pol1, pol2, q и const6__.
- Вычисляет значение выражения, подставляя значения полиномов.
- Формирует новый полином __new_pol = 14.4x^2 – 6.0__.
- Добавляет новый полином __new_pol__ во все таблицы.
- Выводит результат на экран:
     
| Имя     | Полином             |
| :------ | :------------------ |
| new_pol | 14.4x^2 – 6.0       |

4. Примеры ошибок:
- **Ошибка: Полином не найден.** \
  Пользователь вводит выражение **pol3 + pol1**, где **pol3** отсутствует в таблице. \
  Система выводит сообщение об ошибке: **"Ошибка: Полином 'pol3' не найден в таблице."**

# Список объектов и основных алгоритмов
### Объекты: 
1. **Список (List<T>)**
   - Реализация связного списка, используемая для хранения мономов в полиноме.
   - Предоставляет методы для добавления, удаления и доступа к элементам списка.
   - Обеспечивает возможность итерирования по списку.
   - Реализован как шаблонный класс, позволяющий хранить элементы различных типов. 

2. **Моном (Monom)**
   - Элементарный компонент полинома.
   - Хранит коэффициент (coef) и степень (degree).
   - degree кодирует степени x, y, z в одно число (пример: x²y³z → 231).
   
3. **Полином (Polinom)**
   - Алгебраическое выражение как сумма мономов.
   - Хранит мономы в списке List<Monom>.
  
4. **Таблица полиномов (PolinomTable)**
   - Определяет общий интерфейс для работы со всеми типами таблиц.
   - Яляется абстрактным классом. 
   - Содержит методы для добавления, удаления, поиска и получения полиномов.
   - Реализации интерфейса будут соответствовать конкретным типам таблиц (линейная на массиве, на списке, упорядоченная, дерево, хеш-таблица).
     
5. **Конкретные реализации таблиц**
   - **ArrayPolinomTable**: линейная таблица на массиве.
   - **ListPolinomTable**: линейная таблица на списке.
   - **SortedArrayPolinomTable**: упорядоченная на массиве.
   - **AVLTreePolinomTable**: АВЛ-дерево.
   - **RBTreePolinomTable**: красно-черное дерево.
   - **HashOpenAddressingTable**: хеш-таблица с открытой адресацией.
   - **HashChainingTable**: хеш-таблица со списками.

6. **Выражение (Expression)**
   - Представляет собой алгебраическое выражение, записанное в постфиксной форме.
   - Может содержать имена полиномов, константы и операторы.

7. **Стек (Stack<T>)**
   - Используется для хранения операторов (OperatorStack) и операндов (OperandStack) при вычислении выражений.
   - Реализован как шаблонный класс Stack<T>, позволяющий хранить элементы разных типов.

8. **Стек операндов (OperandStack)**
   - Используется при вычислении выражений.
   - Хранит полиномы и промежуточные результаты вычислений.

9. **Стек операторов (OperatorStack)**
   - Используется для преобразования выражения в постфиксную форму с помощью алгоритма сортировочной станции.


### Основные алгоритмы: 
1. **Операции с мономами полинома**
   - **Добавление монома в полином (AddMonom)**:  Добавляет моном в список мономов полинома, поддерживая их упорядоченность по убыванию степени.  Обрабатывает случай сложения коэффициентов при одинаковых степенях.
   - **Разбор монома из строки (ParseMonom)**:  Извлекает коэффициент и степень монома из его строкового представления.  Поддерживает степени переменных x, y и z.

     
2. **Операции над полиномами**
   - **Разбор полинома из строки (ParseFromString)**:  Разбивает строку, представляющую полином, на отдельные мономы и вызывает ParseMonom для каждого из них, добавляя их в полином.
   - **Вычисление значения полинома в точке (calculate)**:  Подставляет значения переменных x, y и z в полином и вычисляет результат
   - **Вывод полинома в строку (operator<<)**:  Преобразует полином в его строковое представление.
   - **Сложение полиномов (operator+)**:  Складывает два полинома, создавая новый полином, содержащий сумму мономов обоих исходных полиномов.  Обрабатывает случаи сложения мономов с одинаковыми степенями.
   - **Вычитание полиномов (operator-)**:  Вычитает один полином из другого.  Аналогично сложению, но с учетом знака.
   - **Умножение полинома на константу (operator\* (double))**:  Умножает все коэффициенты мономов полинома на заданную константу.
   - **Умножение полиномов (operator\* (const Polinom&))**:  Перемножает два полинома, создавая новый полином, содержащий произведение всех мономов исходных полиномов.
   - **Вычисление производной полинома (derivative)**:  Вычисляет производную полинома по заданной переменной (x, y или z).
   - **Вычисление интеграла полинома (integrate)**: Вычисляет интеграл полинома по заданной переменной (x, y или z).

     
3. **Операции со списком (List)**:
      - **Получение размера (size())**: Возвращает количество элементов в списке.
      - **Проверка на пустоту (isEmpty())**: Проверяет, пустой ли список.
      - **Доступ к первому элементу (Front())**: Возвращает ссылку на первый элемент списка.
      - **Добавление в начало (PushFront(const T& val))**: Добавляет элемент в начало списка.
      - **Удаление первого элемента (PopFront())**: Удаляет первый элемент списка.
      - **Доступ по индексу (operator[](size_t pos))**: Позволяет получать доступ к элементам списка по их индексу.

4. **Обработка выражений**
      - **Преобразование выражения в постфиксную форму (алгоритм сортировочной станции)**:  Преобразует алгебраическое выражение из инфиксной формы (обычная запись) в постфиксную форму.
      - **Вычисление выражения в постфиксной форме**:  Вычисляет значение выражения, записанного в постфиксной форме.  Использует стек операндов для хранения полиномов и промежуточных результатов.
  
        
5. **Операции над таблицами**:
      - **Запись элемента в таблицу**: Добавляет полином в выбранную таблицу. Реализация зависит от типа таблицы.
      - **Удаление элемента из таблицы**: Удаляет полином из выбранной таблицы по его имени. Реализация зависит от типа таблицы.
      - **Поиск элемента в таблице**: Ищет полином в выбранной таблице по его имени. Реализация зависит от типа таблицы.
      - **Вывод таблицы на экран**: Выводит содержимое выбранной таблицы на экран в заданном формате.

        
6. **Обработка ошибок**:  Необходимо предусмотреть обработку различных ошибок, которые могут возникнуть в процессе работы программы (например, деление на ноль, отсутствие полинома в таблице, некорректный формат выражения и т.д.).
